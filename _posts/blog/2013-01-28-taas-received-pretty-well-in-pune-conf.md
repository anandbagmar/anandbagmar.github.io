As mentioned in my [earlier post](</blog/taas-test-as-a-service-coming-to-pune/>), I spoke about **_[TaaS - Test-as-a-Service](</blog/tags/#taas>)_** in [UNICOM's Next Generation Testing Conference](<http://www.unicomlearning.com/event/seminar/next-generation-testing-conference-pune>) in Pune on 23rd Jan. The talk went pretty well. Since this talk was for a slightly smaller duration, I had to use a scaled down version of the [**_same slides_**](<http://www.slideshare.net/abagmar/integration-testing-in-enterprises-using-taas-test-as-a-service>) that I used in the [_TaaS talk in Mumbai_](</blog/end-2-end-automated-integration-testing-using-taas-test-as-a-service/>). The _audio_ playback of the talk in Pune is available [**_here_**](<http://db.tt/4SyAmzkH>).  
  
There were a lot of interesting questions that came up this time:  
1\. Can I use TaaS to test in the cloud?  
2\. Can I use TaaS to do load / performance testing?  
3\. What are the security implications of using TaaS?   
4\. Can I use TaaS to do mobile testing across different devices?  
  
I will try to address these questions in subsequent posts.
